Common JSON Errors Explained (With Examples)

JSON is one of the most widely used data formats in modern web development. However, even small syntax mistakes can cause a JSON parse error or an invalid JSON format message.

This guide explains the most common JSON errors, why they happen, and how to fix them quickly.

1. Missing Commas Between Properties

JSON requires commas between key-value pairs inside objects.

Incorrect

{
  "name": "John"
  "age": 30
}

Correct

{
  "name": "John",
  "age": 30
}

Forgetting a comma is one of the most frequent JSON syntax errors.

2. Trailing Commas

Unlike JavaScript objects, JSON does not allow trailing commas.

Incorrect

{
  "name": "John",
  "age": 30,
}

Correct

{
  "name": "John",
  "age": 30
}

Trailing commas often cause an “Unexpected token }” error.

3. Using Single Quotes Instead of Double Quotes

JSON requires double quotation marks for both keys and string values.

Incorrect

{
  'name': 'John'
}

Correct

{
  "name": "John"
}

Single quotes are valid in JavaScript, but not in JSON.

4. Unquoted Keys

Incorrect

{
  name: "John"
}

Correct

{
  "name": "John"
}

All keys must be strings enclosed in double quotes.

5. Invalid Data Types

JSON supports only these data types:

  • String
  • Number
  • Boolean
  • Object
  • Array
  • null

Incorrect

{
  "value": NaN
}

Values such as NaN, Infinity, and undefined are not valid in JSON.

6. Unexpected Token Errors

If you see an error like: 

Unexpected token } in JSON at position 45

This usually means:

  • Missing comma
  • Extra comma
  • Unclosed bracket
  • Invalid quotation mark

Check the exact character position mentioned in the error message.

7. Unclosed Brackets or Braces

Incorrect

{
  "user": {
    "name": "John"
}

Correct

{
  "user": {
    "name": "John"
  }
}

Every opening { must have a closing }, and every [ must have a closing ].

How to Debug JSON Quickly

Here are simple steps to fix JSON parse errors:

  • Format the JSON to visualize structure
  • Look for missing commas
  • Check quotation marks
  • Validate before sending to APIs
